### Metric definitions (Section II.F)
- **RMSE** = √( (1/N)·Σ(ŷ_i − y_i)² ) — Root Mean Squared Error
- **MAE**  = (1/N)·Σ|ŷ_i − y_i| — Mean Absolute Error
- **MAPE** = (100/N)·Σ|ŷ_i − y_i|/|y_i| — Mean Absolute Percentage Error (%)
- **R²**   = 1 − Σ(y_i − ŷ_i)² / Σ(y_i − ȳ)² — Coefficient of determination

The metric functions are implemented in `train.py` → `rmse()`, `mae()`, `mape()`, `r_squared()`.

## Architecture Parameters – Proposed LSTM (Section IV.E)

The `ProposedLSTM` class in `models.py` implements the 5-layer architecture:

| Layer | Description | Parameters |
|-------|-------------|-----------|
| 1 – Contextual Embedding | Linear(input+5, 256) + ReLU | Cyclic time: sin/cos hour, sin/cos weekday, holiday |
| 2 – Multi-Resolution BiLSTM | 3 × BiLSTM(256, 128 units, ds=1,2,4) | ~1.2 M params |
| 3 – Resolution Attention Fusion | Softmax attention + Linear(256,256) + BatchNorm + ReLU | ~0.07 M params |
| 4 – Enc-Dec + Bahdanau Attention | BiLSTM encoder (128→256) + LSTM decoder (256) | ~2.1 M params |
| 5 – MC Dropout + Dual FC Heads | fc_mean(512,1) + fc_logvar(512,1) + Dropout(0.3) | ~0.001 M params |

Total: ~4.55 M trainable parameters (for input_size≈9, hidden_size=256).

## Known Inconsistencies Resolved

| Issue | Original state | Fixed state |
|-------|---------------|-------------|
| SARIMA rolling evaluation | Fit SARIMA N_test times → >4 h runtime | Fit once, forecast all steps (one-shot) |
| ProposedLSTM fc_mean/fc_logvar output dim | `output_size` (bug for horizon>1) | `1` (correct: 1 scalar per decoder step) |
| Early stopping patience | 20 epochs (benchmark) | 40 epochs (matches §VII.A spec) |
| ProposedLSTM in Table I | Missing from benchmark | Added as primary proposed model |
| Benchmark model for multi-horizon/cross-dataset | AttentionLSTM | ProposedLSTM (the full proposed model) |
| GPU acceleration | No AMP, no pin_memory | AMP (bfloat16) + pin_memory for RTX GPUs |
